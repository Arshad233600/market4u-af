
# راهنمای نهایی انتشار در Azure (Secure Deployment Guide) 🚀

این راهنما برای انتشار نسخه **Frontend (PWA)** با رعایت کامل اصول امنیتی تهیه شده است.

---

## ۱. منابع مورد نیاز در Azure (Azure Resources)

برای انتشار این پروژه در فاز فعلی (Demo/MVP)، شما فقط به دو منبع زیر در پنل Azure نیاز دارید:

1.  **Resource Group (گروه منابع):**
    *   یک ظرف (Container) منطقی برای مدیریت منابع.
    *   *مثال نام:* `Market4U-RG`
    *   *ریجن پیشنهادی:* `UAE North` (دبی) یا `Central India` (به دلیل نزدیکی به افغانستان).

2.  **Static Web App:**
    *   سرویس میزبانی فایل‌های استاتیک و React.
    *   *پلن:* **Free** (برای این فاز کاملاً کافی است و SSL رایگان دارد).
    *   *مثال نام:* `market4u-app`

*(هیچ دیتابیس یا سرور دیگری در این مرحله لازم نیست، زیرا ما از داده‌های Mock استفاده می‌کنیم.)*

---

## ۲. چک‌لیست تنظیمات (Configuration Checklist)

بعد از ساخت Static Web App، به بخش **Settings > Configuration** بروید و تنظیمات زیر را انجام دهید:

### ✅ متغیرهای محیطی (Environment Variables)

تنها متغیری که **باید** اضافه کنید:

| نام متغیر (Key) | مقدار (Value) | توضیح |
| :--- | :--- | :--- |
| `VITE_USE_MOCK_DATA` | `true` | فعال‌سازی حالت دمو و دیتابیس مرورگر |

### ❌ هشدار امنیتی (Security Warning)

*   **هرگز** کلیدهای حساس مانند `VITE_API_KEY` (برای جمینای) را در تنظیمات Static Web App قرار ندهید. این کار باعث می‌شود کلید شما در مرورگر کاربران قابل مشاهده باشد.
*   **رفتار صحیح:** در این نسخه دمو، وقتی کلید API وجود نداشته باشد، بخش‌های هوش مصنوعی (تولید توضیحات آگهی) به صورت خودکار غیرفعال می‌شوند و پیامی مبنی بر "نسخه نمایشی" نشان می‌دهند. این امن‌ترین حالت برای انتشار عمومی است.

---

## ۳. دستورالعمل انتشار (Deployment Steps)

1.  کد را به GitHub منتقل کنید.
2.  در پورتال Azure، یک **Static Web App** جدید بسازید.
3.  به GitHub خود متصل شوید و مخزن (Repository) را انتخاب کنید.
4.  در بخش **Build Details**:
    *   **Build Preset:** `Custom` (یا `React`)
    *   **App Location:** `/`
    *   **Output Location:** `dist` (مهم: خروجی بیلد Vite پوشه dist است)
    *   **API Location:** (خالی بگذارید)
5.  دکمه **Create** را بزنید.

پس از چند دقیقه، سایت شما با HTTPS فعال خواهد شد.

---

## ۴. گام بعدی (فاز ۲)

برای فعال‌سازی قابلیت‌های واقعی (هوش مصنوعی، دیتابیس، لاگین):
1.  ساخت **Azure Functions** (به عنوان Backend امن).
2.  انتقال کلید Gemini به **Azure Key Vault**.
3.  اتصال Frontend به Function App.

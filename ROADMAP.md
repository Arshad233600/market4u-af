
# نقشه راه توسعه بازار افغانستان (Bazar Afghanistan Roadmap) 🇦🇫

این سند شامل نیازمندی‌های فنی، زیرساختی و عملیاتی برای تبدیل نسخه فعلی (MVP) به یک محصول نهایی و قابل عرضه در بازار است.

## 🛑 فاز ۱: زیرساخت و بک‌اند (اولویت بسیار بالا)
*در حال حاضر اپلیکیشن از داده‌های ساختگی (Mock) استفاده می‌کند.*

- [ ] **توسعه API Server:**
    - انتخاب تکنولوژی: ASP.NET Core 8 (پیشنهادی برای Azure) یا Node.js.
    - پیاده‌سازی Endpoints برای CRUD آگهی‌ها، کاربران و پیام‌ها.
- [ ] **پایگاه داده (Database):**
    - طراحی اسکیما (Schema) برای Users, Products, Categories, Chats, Transactions.
    - پیشنهاد: PostgreSQL (ارزان و قدرتمند) یا Microsoft SQL Server.
- [ ] **احراز هویت (Authentication):**
    - پیاده‌سازی سیستم ورود با شماره موبایل (OTP) مخصوص افغانستان.
    - اتصال به پنل‌های SMS محلی.
- [ ] **مدیریت فایل (File Storage):**
    - اتصال واقعی به Azure Blob Storage یا AWS S3 برای ذخیره تصاویر.
    - پیاده‌سازی فشرده‌سازی نهایی سمت سرور (Server-side compression).

## 💬 فاز ۲: ارتباطات و تعامل (Real-time)

- [ ] **چت واقعی (Socket):**
    - پیاده‌سازی WebSocket (با SignalR یا Socket.io) برای ارسال پیام آنی.
    - ذخیره تاریخچه چت در دیتابیس.
    - قابلیت ارسال موقعیت مکانی و پیام صوتی در چت.
- [ ] **اعلان‌ها (Notifications):**
    - استفاده از Firebase Cloud Messaging (FCM) برای ارسال Push Notification به مرورگر و موبایل.
    - اطلاع‌رسانی هنگام دریافت پیام جدید یا تغییر وضعیت آگهی.

## 🛡️ فاز ۳: اعتماد و امنیت (حیاتی برای بازار افغانستان)

- [ ] **پنل مدیریت (Admin Panel):**
    - داشبورد برای بررسی و تایید/رد آگهی‌ها قبل از انتشار.
    - سیستم مدیریت گزارش‌های تخلف (Report Management).
    - احراز هویت دستی فروشندگان (بررسی عکس تذکره).
- [ ] **امنیت سایبری:**
    - جلوگیری از حملات DDoS.
    - Rate Limiting (محدودیت تعداد درخواست) برای جلوگیری از اسپم.
    - Sanitization ورودی‌ها برای جلوگیری از XSS و SQL Injection.

## 💰 فاز ۴: مالی و درآمدزایی

- [ ] **کیف پول واقعی:**
    - پیاده‌سازی منطق دفتری (Ledger) برای تراکنش‌ها.
    - قابلیت "پرداخت امن" (بلوکه کردن پول تا زمان تحویل کالا).
- [ ] **درگاه پرداخت:**
    - اتصال به HesabPay، M-Paisa یا درگاه‌های بانکی افغانستان.
- [ ] **پلن‌های درآمدی:**
    - فعال‌سازی واقعی پرداخت برای "نردبان" و "آگهی فوری".

## 🚀 فاز ۵: پرفورمنس و سئو (SEO)

- [ ] **SEO:**
    - پیاده‌سازی Server-Side Rendering (SSR) یا Dynamic Rendering برای ایندکس شدن آگهی‌ها در گوگل.
    - ایجاد Sitemap.xml و Robots.txt داینامیک.
- [ ] **PWA پیشرفته:**
    - کش کردن (Caching) هوشمندانه آگهی‌ها برای دسترسی آفلاین.
    - قابلیت Background Sync برای ارسال پیام‌ها وقتی اینترنت وصل شد.

## 📱 فاز ۶: اپلیکیشن موبایل (Native)

- [ ] خروجی گرفتن از پروژه با استفاده از **Capacitor** یا **TWA (Trusted Web Activity)** برای انتشار در گوگل پلی استور به عنوان اپلیکیشن اندروید.
